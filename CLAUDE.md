# CodePilot 工程规范

## 1. 面向对象（OOP）与类设计

- 所有业务逻辑必须封装在 **结构体（struct）+ 方法** 中，禁止包级全局函数（package-level function）。
- 每个源文件最多允许一个包级函数作为**构造函数**（`New*`），其余所有函数必须是结构体的方法。
- 禁止工具函数（utility）包中存在全局函数——工具函数应作为相关结构体的方法，或定义为私有辅助方法。

## 2. 接口优先

- 通用 API 必须定义为 **Go 接口（interface）**，并附带具体的结构体实现。
- 接口定义与实现分离：接口放在调用方包或独立的 `types` / `contracts` 包中，实现放在具体功能包中。
- 依赖注入优先：结构体通过构造函数接收接口依赖，禁止在方法内部直接实例化具体类型。

## 3. 禁止全局状态与函数堆砌

- 禁止包级可变状态（`var` 声明的全局变量），配置和状态必须通过结构体字段传递。
- 禁止在同一包中堆砌多个不相关的全局函数——每个功能单元必须是独立的 struct + interface。
- 回调、闭包、goroutine 内的匿名函数不受此限，但匿名函数超过 5 行应提取为命名方法。
